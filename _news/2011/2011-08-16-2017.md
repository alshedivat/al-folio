## omicron sdk


Link: [http://code.google.com/p/omicron-sdk](http://code.google.com/p/omicron-sdk)

omicron sdk

![image](/images/omicron_logo1.png-srcw.jpg){:style="max-width: 100%"}
Credit: A. Nishimoto, EVL

omicron sdk is an input abstraction and utility library for virtual reality systems, providing access to a variety of input devices, mostly used in immersive installations and stereo display systems.<br><br>

Some of the devices currently supported by omicron are:<br>
<ul>
<li>All motion capture systems supporting the VRPN protocol</li>
<li>NaturalPoint trackers (TrackIR, Optitrack)</li>
<li>Wii controllers</li>
<li>Xbox 360 controllers</li>
<li>Microsoft Kinect (multiple Kinects supported. omicron also offers some functions for Multi-kinect transform calibration)</li>
<li>SAGE pointer connections</li>
<li>PQLabs multitouch overlays</li>
<li>iPad touch interfaces and dynamic GUIs through the custom Porthole protocol</li>
<li>Thinkgear brainwave interfaces</li>
</ul><br>

omicron abstracts input using modular event services, each providing access to a specific input device. Events can be easily streamed over the network using the omicron connector API. Also, Event services can be chained together to provide advanced functionality.<br><br>

In addition to event services, omicron provides additional utility APIs to simplify the developement of the non-graphics part of VR applications. These APIs are mostly lightweight wrappers on top of well established open source libraries. omicron includes:<br>
<ul>
<li>A configuration file reading system (libconfig)</li>
<li>An xml reading / writing API (<a href='http://www.grinninglizard.com/tinyxml2/index.html'>tinyxml-2</a>)</li>
<li>Multithreading support (<a href='http://tinythread.sourceforge.net'>tinythread++</a>)</li>
<li>A simple tcp client / server API (<a href='http://think-async.com'>asio</a>)</li>
<li>A mathematical / geometry library (<a href='http://eigen.tuxfamily.org/index.php?title=Main_Page'>eigen</a>)</li>
</ul><br>

omicron input support can be integrated into C++ applications as a static library, or it can be run as a standalone input server, streaming input data to multiple applications. Omicron comes with client-side interfaces for Unity and Processing.<br><br>

C++, using a single header. Only a few lines of code needed. For more information see the omicronConnector section on the <a href='http://code.google.com/p/omicron-sdk/wiki/GettingStarted'>Getting started</a> page.
